Document Classification — Hardware Options

Deployment options for the local AI document-classification solution
Prepared for client review · June 2026 · Confidential

The solution runs a local AI model to read and classify documents (PDFs, scans, and images) entirely on the customer's own hardware — no cloud, no data leaving the premises. Because all processing is local, the choice of hardware determines speed, the size of model that can run, and the deployment form factor.

This document presents three on-premises hardware options, from an entry-level GPU workstation to a high-end server and an all-in-one mini-PC, so the client can match capability, footprint, and budget. All three run the same validated AI model; they differ in speed, headroom, and cost. A key technical factor throughout is the AI model size the hardware can host: the larger the model, the higher the classification accuracy on difficult or scanned documents — and the more memory the hardware needs.

The options in detail

Option 1 · Entry GPU

NVIDIA RTX 5070 Ti — Workstation

Discrete GPU in a standard workstation or small server
  • Runs the validated 8B vision model with classification in a few seconds per document.
  • Lowest-cost on-prem GPU option that comfortably hosts the tested model.
  • Fits a standard workstation — no dedicated server room required.
  • Trade-off: less headroom than the 5080 for larger / long-context models.
Best value ~4 000 zł (GPU)
Option 2 · Server

NVIDIA RTX 5080 — Server / Workstation

Discrete high-end GPU in a server or workstation chassis
  • Runs the validated 8B vision model with comfortable headroom.
  • Fastest classification; room to grow into larger or longer-context models for higher accuracy.
  • Can serve multiple users / higher document volumes from one central machine.
Client requirement: a proper server with an adequate power supply and cooling for a high-wattage GPU. This is the highest-power option, but also the most capable.
Highest performance Needs server & PSU ~6 000 zł (GPU)
Option 3 · Edge — All-in-one

AMD Ryzen AI Max+ 395

Mini-PC APU with up to 128 GB unified memory (Strix Halo)
  • Up to 128 GB unified memory shared with the GPU — runs the 8B vision model comfortably, and far larger models if higher accuracy is wanted.
  • Delivers near-workstation capability in a small mini-PC — no server chassis, no discrete GPU, moderate power.
  • Self-contained edge box: full capability on-premises without a server room.
  • Trade-off: highest up-front cost of the three, as a complete system.
Full capability, no server Highest cost ~14 000 zł (full system)

A note on model size & accuracy

The hardware's memory caps the size of AI model it can host, and model size is the main lever on classification quality. All three options above run the validated 8B vision model — the configuration that reads both digital text and scanned/image documents reliably (4/4 correct in our proof-of-concept). The differences between them are speed and headroom (the RTX 5080 and the 128 GB Ryzen leave more room for larger, longer-context models should higher accuracy be required later), not whether the tested model fits.

Recommendation

Best value (Option 1 — RTX 5070 Ti, ~4 000 zł): the lowest-cost way to run the validated model on a standard workstation. The right starting point for typical, interactive document volumes.

Most performance / growth (Option 2 — RTX 5080, ~6 000 zł): for higher volumes or to leave room for larger models — provided the client can supply a server with adequate power and cooling.

All-in-one, no server (Option 3 — Ryzen AI Max+ 395, ~14 000 zł): a single self-contained box with the most memory headroom, ideal where a server room is not available or the largest models may be wanted later.
